# 实施版：上线闭环检查清单

更新时间：2026-05-11

| 序号 | 项目 | 当前落地方式 | 发布判定 |
| --- | --- | --- | --- |
| 1 | 干净发布版本 | 本地提交前运行 `git status --short`；CI 运行模板校验、审计、回归、构建、E2E | 工作区无未提交改动，CI 绿色 |
| 2 | 生产配置 | `.env.example` 改为生产模板，`.env.local.example` 承载本地默认值，`verify:production-env` 与 API 启动时运行时校验真实环境 | 无默认 secret、JWT 双密钥不同且长度不少于 32、无默认管理员密码、无 mock provider、无 localhost |
| 3 | 真实 provider | API 启动阶段禁止生产 mock；对象存储、AI、地图 POI 必填变量缺失即失败；AI 支持 OpenAI-compatible；短信默认关闭 | 预发完成上传、AI 任务、地图地点搜索全链路验证；如启用短信需单独联调 |
| 4 | 部署方案 | `deploy/docker-compose.prod.yml` 编排 API/Web/Admin/MySQL/Redis/Nginx；`deploy/nginx` 提供 TLS 反代 | 域名、TLS、健康检查、日志、迁移、回滚流程明确 |
| 5 | 合规文档 | `docs/legal/用户协议.md`、`docs/legal/隐私政策.md`、`docs/legal/儿童信息保护规则.md`；App 关于页展示完整条款摘要 | 上线前补经营主体、联系方式、投诉渠道 |
| 6 | 运维上线项 | 部署手册覆盖备份、日志、审计、健康检查、监控告警、迁移和回滚 | 备份恢复演练通过，告警联系人确认 |

## 必跑命令

```bash
npm run verify:production-template
npm audit --omit=dev --audit-level=moderate
npm run regression:baseline
npm run build:api
npm run build:web
npm run build:admin
npm run test:e2e
```

## App 与后台联调抽检

| 模块 | 抽检点 |
| --- | --- |
| App 登录/注册 | 账号密码登录、邀请码注册、协议勾选、错误提示中文化；邀请码只在注册时使用 |
| App 建档 | 创建孩子、查看孩子资料、家庭成员入口 |
| App 记录 | 创建文字记录、上传图片、详情页媒体预览、AI 任务创建 |
| App 地点 | 发布页地点搜索、浏览器定位授权/拒绝、选择 POI 后保存回显 |
| App 我的 | 个人资料、隐私设置、帮助反馈、关于与协议 |
| 后台登录 | 错误密码中文提示、成功登录、角色展示 |
| 后台总览 | 用户数、孩子数、记录数、媒体数、AI 任务分布、最近审计 |
| 后台列表 | 用户、孩子、记录、媒体、AI 任务、审计日志均支持查询、分页、详情 |
| 后台操作 | 用户冻结/解冻、记录下架/恢复、媒体审核、AI 重试/取消均有确认、失败提示和审计日志 |

## 暂不阻塞但上线前必须人工确认

- 如后续重新启用短信，需确认真实短信签名和模板审核状态。
- 真实对象存储 bucket 权限、CORS 和 CDN 访问策略。
- 真实 AI 服务额度、超时、失败重试和内容安全要求。
- 高德地图 key、配额、服务域名可用性和 POI 搜索结果准确性。
- Android/iOS 真机相机、录音、视频选择、定位权限授权/拒绝流程。
- 生产域名备案、TLS 证书续期和告警联系人。
- 用户协议、隐私政策、儿童信息保护规则中的经营主体与联系方式。

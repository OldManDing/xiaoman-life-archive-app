# 发布前验收标准与执行清单

更新时间：2026-05-27

## 通过判定

发布前验收按 P0 / P1 / P2 分级：

- P0：阻塞上线。必须修复并复验通过，不允许带问题发布。
- P1：影响体验或运营效率。默认应修复；如延期，必须写明原因、影响面和后续 owner。
- P2：上线后优化。不得影响核心链路、安全、数据正确性和付费/合规判断。

最终结论只能是以下三种：

- 通过：全部 P0 通过，P1 无明显风险或已记录延期方案。
- 条件通过：全部 P0 通过，但存在可控 P1 延期项。
- 不通过：任一 P0 未通过，或验收证据不足以证明关键链路可用。

## 验收证据要求

每一项验收必须至少保留一种证据：

- 自动化测试命令与结果。
- 真实浏览器截图或长截图。
- Android / iOS 真机或模拟器截图、录屏、网络响应记录。
- API 请求与响应摘要。
- 后台操作审计日志。

口头确认、只看代码、只看单页截图都不能作为 P0 通过依据。

## P0 验收项

| 编号 | 范围 | 标准 | 最低证据 |
| --- | --- | --- | --- |
| P0-01 | 生产配置 | 生产模板无默认 secret、无 localhost、无 mock provider；生产启动缺关键 provider 必须失败 | `npm run verify:production-template`、生产配置校验 |
| P0-02 | API 健康 | API 可启动，`/api/v1/health` 返回健康状态，数据库连接正常 | API smoke / E2E |
| P0-03 | 注册邀请码 | 后台可生成邀请码；App 注册必须填写有效邀请码；邀请码只用于注册，登录不要求邀请码；邀请码复用失败 | Admin E2E + Auth/API E2E |
| P0-04 | 登录与协议 | App 账号密码登录成功；未勾选协议不能登录；错误账号密码显示“账号或密码错误” | App E2E / 真机截图 |

| P0-13 | 后台账号管理 | 可查看用户详情、登录信息、重置密码入口；冻结/解冻有确认、失败提示和审计 | Admin E2E |
| P0-14 | 后台邀请码管理 | 后台有明确入口生成注册邀请码，生成后一次性展示、可复制，列表可查历史状态 | Admin E2E + 截图 |
| P0-15 | 后台运营列表 | 用户、孩子、记录、媒体、AI 任务、审计日志支持查询、分页、详情；关键操作可恢复 | Admin E2E |

## P1 验收项

| 编号 | 范围 | 标准 |
| --- | --- | --- |
| P1-01 | 搜索 | 搜索入口、历史搜索、热门标签、结果跳转可用，空状态有明确引导 |
| P1-02 | 月报/纪念册 | 页面不是纯占位；有故事化摘要、导出或申领反馈 |
| P1-03 | 地点 | 地点搜索、定位、手动输入均有反馈；真实地图 provider 上线前需复核 |
| P1-04 | 媒体格式 | JPG/PNG/WebP、MP4/WebM/MOV、M4A/MP3/WAV 等格式按设备能力验证 |
| P1-05 | iOS | iOS Safari 或 WebView 验证登录、注册、上传、定位、键盘和安全区 |
| P1-06 | 运维 | npm audit、生产 docker compose config、备份恢复演练、告警联系人确认 |

- 用户反馈工单系统。
- 高级筛选组合。
- 多端深度同步策略。

## 发布前必跑命令

```bash
npm run verify:production-template
npm audit --omit=dev --audit-level=moderate
npm run regression:baseline
npm run build:api
npm run build:web
npm run build:admin
npm run test:e2e
docker compose -f deploy/docker-compose.prod.yml --env-file .env.example config --quiet
```

## 验收执行顺序

1. 配置与工程门禁：生产模板、依赖审计、typecheck、单元测试、构建。
2. API 门禁：健康检查、注册邀请码、登录、注销账号后重新登录失败。
3. App 关键旅程：登录/注册/协议返回、首页、记录、时间轴、家庭、我的、账号注销。
4. 后台关键旅程：登录、总览、账号管理、邀请码、列表详情、操作审计。
5. 视觉与产品审核：桌面、移动端、模拟器长截图；按 P0-17 / P0-18 判定。
6. 复验：所有修复项必须回到对应验收项重新跑证据。

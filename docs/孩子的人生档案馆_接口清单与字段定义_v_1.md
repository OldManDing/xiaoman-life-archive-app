# 1. 文档说明

## 1.1 文档目的
本文档用于定义 MVP 阶段的核心接口清单、请求参数、响应字段、状态码约定与对象结构，作为前后端联调、测试用例编写和接口验收依据。

## 1.2 接口设计原则
1. 统一使用 RESTful 风格。
2. 统一使用 HTTPS。
3. 统一使用 JSON 作为请求与响应格式。
4. 删除操作采用逻辑删除，除非特别说明。
5. 列表接口统一支持分页。
6. 时间字段统一使用 ISO 8601 或时间戳，项目内需统一一种，本文档先以 ISO 8601 字符串示例。

## 1.3 通用前缀
建议统一前缀：

```text
/api/v1
```

示例：

```text
GET /api/v1/children
POST /api/v1/records
```

# 2. 通用约定

## 2.1 通用响应结构

```json
{
  "code": 0,
  "message": "success",
  "request_id": "req_xxx",
  "data": {}
}
```

## 2.2 字段说明
- `code`：业务状态码，0 表示成功，非 0 表示失败
- `message`：响应描述
- `request_id`：请求追踪 ID
- `data`：业务数据体

## 2.3 通用分页结构

```json
{
  "list": [],
  "page": 1,
  "page_size": 20,
  "total": 100,
  "has_more": true
}
```

## 2.4 通用状态码建议

### HTTP 状态码
- 200：请求成功
- 400：参数错误
- 401：未登录或 token 失效
- 403：无权限
- 404：资源不存在
- 409：状态冲突
- 500：服务端异常

### 业务状态码
- 0：成功
- 1001：参数校验失败
- 1002：未登录
- 1003：无权限
- 1004：资源不存在
- 1005：状态不允许
- 1006：上传失败
- 1007：生成失败
- 1008：重复操作
- 1099：系统异常

# 5. 记录接口

# 5.1 创建记录

## 接口
```text
POST /api/v1/records
```

## 请求参数
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| child_id | string | 是 | 孩子 ID |
| record_type | string | 是 | image / video / audio / text / mixed / milestone |
| title | string | 否 | 标题 |
| content_text | string | 否 | 正文 |
| media_list | array | 否 | 媒体列表，结构见 MediaItem |
| tags | array<string> | 否 | 标签列表 |
| event_time | string | 否 | 事件时间，默认当前时间 |
| location_text | string | 否 | 地点 |
| weather_text | string | 否 | 天气，MVP 可预留 |
| visibility_scope | string | 否 | family / custom，MVP 默认 family |
| is_milestone | bool | 否 | 是否里程碑 |
| milestone_template_id | string | 否 | 里程碑模板 ID |
| status | string | 否 | draft / published，默认 published |

## media_list 元素
| 字段 | 类型 | 必填 | 说明 |
|---|---|---:|---|
| media_type | string | 是 | image / video / audio |
| url | string | 是 | 媒体地址 |
| cover_url | string | 否 | 封面图，视频可用 |
| duration | int | 否 | 时长，秒 |
| width | int | 否 | 宽 |
| height | int | 否 | 高 |
| size | int | 否 | 字节数 |

## 业务规则
- `content_text` 与 `media_list` 至少有一项非空。
- `status=draft` 时不进入时间轴、不进入月报统计。
- `title` 为空时可由前端或后端兜底生成。

## 响应 data
返回 `RecordDetail` 对象。

# 12. 接口联调注意事项

## 12.1 前后端统一约定项
1. 时间格式统一。
2. 逻辑删除资源前端默认不可见。
3. 枚举值统一由接口文档维护，不允许前后端各自定义。
4. 权限错误必须返回明确 code 与 message。
5. 月报生成与导出均为异步任务，前端需轮询或订阅状态。

## 12.2 测试重点
1. 参数缺失时是否正确报错。
2. 不同角色下的权限是否正确。
3. 草稿与正式发布状态是否正确分流。
4. 删除记录后时间轴与月报是否同步变化。
5. 导出成功后下载链接是否可用且按时过期。

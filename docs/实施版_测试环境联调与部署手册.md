# 实施版：测试环境联调与部署手册

## 本地基础设施

```bash
docker compose up -d mysql redis minio minio-init
```

默认端口：

- MySQL：`3307`
- Redis：`6379`
- MinIO API：`9000`
- MinIO Console：`9001`

`minio-init` 负责创建本地对象存储 bucket：`xiaoman-archive-local`。如果上传凭证正常生成但文件 PUT 返回 `NoSuchBucket`，先确认该初始化服务已执行成功。

## 环境文件

复制根目录 `.env.example` 为 `.env`，至少检查：

```env
DATABASE_URL=mysql://xiaoman:password@localhost:3307/xiaoman_archive
APP_PORT=3000
JWT_ACCESS_SECRET=replace_me_access_secret
JWT_REFRESH_SECRET=replace_me_refresh_secret
CORS_ORIGINS=http://localhost:5173,http://localhost:5174
```

共享测试环境必须替换默认 JWT secret、管理员初始密码、短信配置和对象存储凭证。

## 启动顺序

```bash
npm run prisma:generate
npm run prisma:dbpush
npm run dev:api
npm run dev:web
npm run dev:admin
```

默认访问地址：

- API：`http://localhost:3000/api/v1`
- 用户端：`http://localhost:5173`
- 管理后台：`http://localhost:5174`

## 联调检查

- `GET /api/v1/health` 返回 `status=ok`
- 用户验证码登录可成功获取 access token 和 refresh cookie
- 创建记录后能在时间轴查看
- 图片上传后创建记录，记录详情 `media_list` 有数据
- 管理后台禁用用户后，用户端未过期 token 请求返回 401
- 管理后台禁用管理员后，后台未过期 token 请求返回 401

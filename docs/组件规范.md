# 组件规范

## 1. 设计风格关键词

- 温柔
- 清爽
- 轻盈
- 亲和
- 有品质感
- 适合家庭类产品
- 不花哨、不廉价

## 2. 颜色建议

### 主色
- 主品牌色：柔和暖色 / 低饱和亲子色
- 主按钮、强调态使用主色

### 辅助色
- 成功：绿色系
- 警告：橙色系
- 错误：红色系
- 信息：蓝灰系

### 中性色
- 标题深色
- 正文中灰
- 辅助浅灰
- 背景浅灰白
- 分割线淡灰

## 3. 字体层级建议

| 层级 | 用途 | 建议字号 |
|---|---|---|
| H1 | 页面主标题 | 24 |
| H2 | 模块标题 | 20 |
| H3 | 卡片标题 | 18 |
| Body1 | 正文主要内容 | 16 |
| Body2 | 正文辅助内容 | 14 |
| Caption | 辅助说明 / 时间 | 12 |

## 4. 圆角规范

- 大卡片：20-24
- 中卡片 / 弹窗：16-20
- 输入框 / 按钮：12-16
- 标签：999 圆角胶囊或 10-12

## 5. 间距规范

- 页面左右边距：16-20
- 模块间距：16 / 20 / 24
- 卡片内边距：12 / 16 / 20
- 表单项垂直间距：12-16

## 6. 按钮规范

### 主按钮
- 用于提交、确认、主路径入口
- 必须具备 normal / pressed / disabled / loading

### 次按钮
- 用于次级操作

### 危险按钮
- 用于删除类操作
- 必须二次确认

## 7. 表单规范

- 输入框高度统一
- 标签与输入区域对齐
- 错误提示显示在字段下方
- 必填标识清晰但不过度刺眼

## 8. 列表 / 时间轴规范

- 记录卡片信息层级清晰
- 时间信息弱化但可识别
- 缩略图比例统一
- 标签和分类不应喧宾夺主

## 9. 空状态规范

每个空状态必须包含：
- 插图 / 图标
- 标题
- 说明文案
- 主行动按钮（如适用）

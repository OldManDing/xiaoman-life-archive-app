# 后台 UI 与 API 开发任务（2026-05-10）

## 当前目标

围绕“年轮”管理后台补齐运营工作台体验，并明确后续 API 开发顺序。后台需要保持全中文界面，所有角色、状态、类型、任务结果都必须通过中文标签展示。

## 已落地的 UI 基础

- 左侧固定导航：总览、用户运营、孩子档案、成长记录、媒体库、AI 任务、审计日志。
- 右侧工作区：当前环境、中文界面标识、页面标题、说明和模块内容。
- 总览页：模块卡片、开发任务入口、后台 UI 设计方向。
- 列表页：统一查询面板、结果总数概览、中文状态标签、分页面板。
- 中文化：会员、用户状态、孩子状态、性别、记录类型、记录状态、可见范围、媒体类型、AI 任务状态、后台角色、审计动作。

## 后台 UI 待开发

1. 用户详情抽屉
   - 展示用户基础资料、手机号、会员信息、最近登录、关联孩子、关联家庭。
   - 支持冻结/解冻原因填写和操作确认。

2. 孩子档案详情抽屉
   - 展示孩子资料、家庭成员、档案状态、最近记录。
   - 支持运营备注和档案状态查看。

3. 成长记录详情与审核
   - 展示标题、正文、标签、媒体、AI 摘要、创建者、可见范围。
   - 支持下架、恢复、查看媒体和查看 AI 任务。

4. 媒体库审核界面
   - 支持图片预览、文件信息、归属关系、上传者信息。
   - 支持审核通过、标记异常、下架。

5. AI 任务管理
   - 展示任务输入、输出、失败原因、重试次数。
   - 支持重试、取消、查看关联记录。

6. 审计日志详情
   - 展示操作者、动作、目标对象、IP、User-Agent、发生时间。
   - 支持按动作、对象类型、时间筛选。

## API 待开发

1. 后台总览 API
   - `GET /api/v1/admin/dashboard`
   - 返回用户数、孩子数、记录数、媒体数、AI 任务状态分布、最近审计日志。

2. 后台详情 API
   - `GET /api/v1/admin/users/:user_no`
   - `GET /api/v1/admin/children/:child_no`
   - `GET /api/v1/admin/records/:record_no`
   - `GET /api/v1/admin/media/:media_no`
   - `GET /api/v1/admin/ai-jobs/:job_no`

3. 后台审核/运营操作 API
   - `PATCH /api/v1/admin/records/:record_no/status`
   - `PATCH /api/v1/admin/media/:media_no/status`
   - `POST /api/v1/admin/ai-jobs/:job_no/retry`
   - `POST /api/v1/admin/ai-jobs/:job_no/cancel`

4. 审计增强 API
   - 列表支持动作、目标类型、时间范围筛选。
   - 所有后台写操作记录操作人、目标对象、原因和请求来源。

## 开发顺序

1. 先完成后台详情 API 和详情抽屉，解决“列表只能看概要”的问题。
2. 再完成记录/媒体审核操作，形成最小运营闭环。
3. 再完成 AI 任务重试/取消，形成异步任务处理闭环。
4. 最后补 dashboard 真实统计数据和审计筛选。

## 验收标准

- 后台界面不直接展示英文枚举值。
- 所有列表页都有查询、总数、分页、中文状态。
- 详情抽屉能从列表直接打开，不跳失当前上下文。
- 后台写操作都必须有确认、失败提示和审计记录。
- `npm run typecheck:admin`、`npm run test:admin`、`npm run build:admin` 必须通过。

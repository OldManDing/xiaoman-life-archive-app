# 实施版：回归验证基线

## 目标

在每轮交付后用同一组命令确认 API、用户端和管理后台没有明显回归。

## 推荐命令

```bash
npm run prisma:generate
npm run typecheck:api
npm run test:api
npm run smoke:test:api
npm run typecheck:web
npm run test:web
npm run typecheck:admin
npm run test:admin
```

完整基线可直接执行：

```bash
npm run regression:baseline
```

## 手工验收路径

1. 登录 / 注册 -> 完善孩子档案 -> 首页
2. 首页 -> 新增记录 -> 上传图片 -> 发布 -> 详情媒体数量大于 0
3. 时间轴 -> 详情 -> 编辑 -> 保存成功
4. 家庭页 -> 邀请成员 -> 成员管理
5. 我的页 -> 未开放能力显示“规划中/待接入”，不出现可保存的假设置
6. 管理后台 -> 用户列表 -> 禁用用户后确认用户端访问被拒绝

## 当前风险

- 视觉验收仍需在真实浏览器里对照 Figma 截图做最终检查。
- 外部短信、存储和 AI provider 尚未进入生产级联调。

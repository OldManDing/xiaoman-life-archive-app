# 实施版：回归验证基线

更新时间：2026-05-28

## 目标

在每轮交付后用同一组命令确认 API、用户端和管理后台没有明显回归。

## 推荐命令

```bash
npm run prisma:generate
npm run typecheck:api
npm run test:api
npm run smoke:test:api
npm run typecheck:web
npm run test:web
npm run typecheck:admin
npm run test:admin
```

完整基线可直接执行：

```bash
npm run regression:baseline
```

## 手工验收路径

1. 登录 / 注册 -> 完善孩子档案 -> 首页
2. 首页 -> 新增记录 -> 上传图片 -> 发布 -> 详情媒体数量大于 0
3. 时间轴 -> 详情 -> 编辑 -> 保存成功
4. 家庭页 -> 邀请成员 -> 成员管理
5. 我的页 -> 月报、纪念册、导出、帮助反馈、隐私、账号安全入口可进入；不得出现“规划中/待接入”等发布前占位表达
6. 管理后台 -> 用户、家庭、孩子、记录、媒体、内容风险、客服反馈、档案交付、套餐权益、系统配置和审计队列可查询或操作
7. 管理后台 -> 用户列表 -> 禁用用户后确认用户端访问被拒绝

## 当前风险

- 高德真实 POI 已使用生产 Web 服务 Key 通过 `verify:live-readiness`，报告中 `poi` 返回 `source=amap` 的文本 POI 候选；后续更换 key、provider 或服务器出口策略时必须重新复验。
- iOS Safari / WebView、Android 真机权限、对象存储签名上传和外部 provider 探针仍需按发布前清单保留设备或线上证据。
- 每轮 release candidate 仍必须重新执行 `regression:baseline`、三端构建、Playwright E2E、生产模板、安全基线和 live readiness；历史通过记录不能替代当前构建证据。

- [ ] `npm run regression:baseline` 已通过或失败项已有明确说明
- [ ] 管理员初始账号策略已确认，生产环境不允许使用默认弱口令
- [ ] 用户协议、隐私政策和儿童信息保护说明已补齐

## 2026-05-28 补充要求

- `live-readiness-latest.json` 只能作为当前 provider 的证据使用：后台 `ops-readiness` 会校验报告中的 `providers.ai` / `providers.map` 与当前运行配置一致，不一致时必须重新执行带测试账号的 `npm run verify:live-readiness`。
- 修复 AI 或地图配置后，不能沿用旧 passed 报告；必须保留新的 `artifacts/app-live-audit/live-readiness-latest.json`，并确认 `aiPreview` 与 `poi` 两个子检查同时通过。
- AI provider 如后续更换 endpoint/model/key，不能只看健康检查 provider 名称；必须重新通过 `ai-jobs/preview` 线上调用和 `verify:live-readiness` 后才能宣称完成。
- `LIVE_READINESS_ALLOW_P1_DEFERRALS=1` 只用于 P1/P2 已记录延期项的条件通过，不允许掩盖 P0 失败；高德 Key 申请完成后必须去掉该开关并重新复验。
